# 마크다운 → PDF 변환 정적 웹페이지 — 설계 문서

- **작성일**: 2026-06-24
- **상태**: 설계 확정 (구현 전)

## 1. 목적

마크다운(.md) 파일을 업로드하면 브라우저에서 렌더링해 미리보고, 브라우저 네이티브 인쇄로 고품질 PDF(텍스트 선택 가능·벡터·한글 안정)를 저장하는 **순수 정적 웹페이지**.

주 사용처: 임용 스터디 노트(한글, 수식·다이어그램·이미지 포함)를 PDF로 변환.

## 2. 핵심 결정 사항

| 항목 | 결정 |
|---|---|
| PDF 생성 방식 | 브라우저 네이티브 인쇄(`window.print()` → "PDF로 저장") + Paged.js |
| 입력 방식 | 파일 업로드 → 변환 (라이브 에디터 없음) |
| 파일 개수 | 여러 .md 업로드 → 목록에서 하나 선택 → 각각 인쇄 |
| 화면 레이아웃 | 상단 툴바(설정 접힘) · 왼쪽 파일 목록 · 넓은 미리보기 |
| 이미지 | .md와 이미지 파일/폴더를 **함께 업로드**, 파일명으로 매칭 |
| 수식 | KaTeX (`$...$`, `$$...$$`) |
| 다이어그램 | Mermaid |

## 3. 아키텍처

- **순수 정적 사이트**: 빌드 도구 없음. `index.html` + `style.css` + `app.js`. 라이브러리는 CDN 로드.
- **서버 없음**: 모든 처리가 브라우저 내에서 완결 → 업로드 파일이 외부로 전송되지 않음(프라이버시).
- 배포: GitHub Pages / Netlify / Vercel 등 정적 호스팅.
- **권장 브라우저**: Chromium 계열(Chrome/Edge). Paged.js 인쇄가 가장 안정적. 안내 문구 노출.

### 파일 구조
```
index.html      # 마크업 + CDN <script>/<link>
style.css       # 화면 UI + @media print + @page
app.js          # 파일 처리 · 변환 파이프라인 · 설정 · 인쇄 제어
```

## 4. 라이브러리 스택

| 역할 | 라이브러리 | 비고 |
|---|---|---|
| MD 파싱 | **markdown-it** | GFM 표/체크박스, 각주 플러그인 |
| 수식 | **markdown-it-texmath** + **KaTeX** | 파싱 단계에서 수식 토큰화 (markdown이 `_`/`*`를 망가뜨리지 않도록) |
| XSS 방지 | **DOMPurify** | SVG·MathML·KaTeX 클래스 허용 설정 |
| 코드 하이라이팅 | **highlight.js** | |
| 다이어그램 | **Mermaid.js** | `securityLevel: 'strict'` |
| 인쇄 페이지네이션 | **Paged.js** | @page 여백 · 페이지 번호 · 머리말 |
| 한글 폰트 | **Pretendard** (+ 대체 Noto Sans KR) | CDN, 인쇄 전 로드 보장 |

## 5. 화면 (레이아웃)

- **상단 툴바**: 로고 · `＋파일 추가` · `⚙설정 ▾`(펼침 패널) · `🖨 인쇄`
- **왼쪽 사이드바**: 업로드한 .md 목록 (선택 / 삭제, 선택 항목 ✓ 표시)
- **메인**: 선택된 파일의 A4 미리보기 (Paged.js 페이지 단위 렌더)
- `@media print`에서 툴바·사이드바 숨김, 미리보기만 인쇄.

## 6. 설정 패널 (localStorage 저장, 변경 시 디바운스 후 즉시 반영)

- **① 페이지**: 용지(A4/Letter) · 방향(세로/가로) · 여백(mm)
- **② 글꼴/본문**: 본문 폰트(Pretendard 등) · 글자 크기(pt) · 줄간격
- **③ 코드**: 하이라이트 테마
- **④ 머리말/페이지 번호**: ☑ 페이지 번호(하단 가운데) · ☐ 문서 제목 머리말
- `기본값으로` 버튼.
- 설정 변경 → 디바운스(예: 300ms) → Paged.js 재페이지네이션(과도한 재계산 방지).

## 7. 변환 파이프라인 (순서 고정)

```
1. 파일 읽기 (FileReader) — .md 텍스트 + 함께 업로드한 이미지(Blob) 맵 구성
2. markdown-it(+texmath) 파싱 → HTML  (수식은 이 단계에서 KaTeX HTML로)
3. DOMPurify 정화 (SVG·MathML·KaTeX 허용)
4. 상대경로 <img> 매칭: 업로드 이미지 맵에서 파일명으로 찾아 src를 ObjectURL로 치환
5. DOM에 주입
6. highlight.js 코드 하이라이팅
7. Mermaid 렌더 (```mermaid 블록 → SVG)   ← 비동기 완료 대기
8. document.fonts.ready 대기              ← 한글 폰트 로드 보장
9. Paged.js 페이지네이션 → 미리보기 표시
```

> 순서 근거: Paged.js는 실제 렌더 크기로 페이지를 나누므로, Mermaid/KaTeX/폰트가 **모두 렌더된 뒤** 마지막에 실행해야 한다.

## 8. 인쇄 흐름

- `🖨 인쇄` → `window.print()` → 브라우저 인쇄창 → 사용자가 **"PDF로 저장"** 선택.
- **여백 전담**: Paged.js가 페이지 여백을 만들고, 인쇄용 `@page { margin: 0 }`으로 이중 적용 방지.
- 페이지 번호: CSS 카운터 / Paged.js. 머리말: Paged.js margin box.
- `print-color-adjust: exact`로 코드 배경색·하이라이트 유지.
- 여러 파일은 **하나 선택 → 인쇄**를 반복(네이티브 인쇄 특성상 파일별 개별 저장).

## 9. 페이지 나눔(인쇄 품질)

- `pre`, `table`, KaTeX 디스플레이 블록, Mermaid SVG: `break-inside: avoid`.
- 제목(h1~h3): `break-after: avoid` (제목만 페이지 끝에 고아로 남지 않게).
- 한 페이지보다 큰 Mermaid/이미지: `max-width: 100%`로 폭에 맞춰 축소.

## 10. 이미지 처리 (함께 업로드)

- 업로드 영역은 .md와 이미지 파일을 함께 받음. **폴더 드래그**(`webkitdirectory` / DataTransfer entries)도 지원해 상대경로 구조 보존.
- 렌더 시 상대경로 `<img src>`의 파일명(basename)을 업로드 이미지 맵과 대조 → `URL.createObjectURL(blob)`로 치환.
- `http(s)://` 이미지는 그대로 사용. 매칭 실패 시 "이미지 없음" 플레이스홀더 표시.

## 11. 예외 상태

- .md 아닌 파일 업로드: 목록에서 무시 또는 경고.
- 빈 .md: "내용 없음" 안내.
- Mermaid 문법 오류: 해당 블록에 에러 메시지 표시(전체 렌더는 계속).
- 비-Chromium 브라우저: 인쇄 품질 안내 배너.

## 12. 범위 밖 (YAGNI)

- 라이브 마크다운 에디터
- 여러 .md를 한 PDF로 합치기
- 서버사이드 변환 / 자동 일괄 다운로드
- 클라우드 저장·로그인

## 13. 미해결/구현 시 검증 필요

- Paged.js와 Mermaid SVG(foreignObject·중복 ID) 호환성 — 실제 렌더로 확인.
- 대용량 노트(다이어그램 다수) 페이지네이션 성능.
